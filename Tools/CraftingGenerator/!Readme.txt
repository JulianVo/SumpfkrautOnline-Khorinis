Starthinweis:
-Zur Ausführung ist .NET Framework 4.5 notwendig
-Datenbank muss sich auf localhost befinden
-Es muss eine Datenbank mit Name 'gothic_multiplayer' existieren

-Per Doppelklick in die jeweilige Box ein Item erstellen und auswählen
-->ACHTUNG: Box reagiert auch auf Doppelklick, wenn bereits das Erstellen eines neuen Items "gestartet" wurde

-Zum Schluss per Commit in die Datenbank übertragen (Die Textbox unter "Protokoll" sollte entspr. Erfolgsmeldung liefern)
-In der Datei "Ausgabe_SQL.txt" wird der zugehörige SQL-Code (für Wiederverwendung) erzeugt.


Anmerkungen:
-Um die Rezepte während des Spiels neu zu laden - Befehl: /reload
-Die Dateie anims.txt sollten selbsterklärend sein

WICHTIG: Die verfügbaren Items und Werkzeuge werden aus der Datenbank geladen. Also ist eine Datenbank wie auf dem Server unabdingbar.
